# AlphaForge

## Overview

AlphaForge is a local research workflow engine for authorized WorldQuant / BRAIN research assistance, focused on iterative alpha generation, backtest workflow orchestration, runtime observability, memory-compatible refinement, and release-safe automation.

This repository is intended for authorized research workflow assistance only:

- It does not include account credentials, passwords, API keys, private Alpha results, or platform-owned proprietary data.
- It does not perform Submit operations by default.
- Users are responsible for complying with applicable laws, institutional policies, and platform terms of service.
- The project is not affiliated with, endorsed by, or sponsored by WorldQuant, WorldQuant BRAIN, or any related platform operator.

Licensed under GPL-3.0-only. See [LICENSE](LICENSE).

The internal Python package remains `wq_workflow` for compatibility. The main entrypoint remains `worldquant_auto_workflow.py`.

## Current Stable Path

### Refactored Pipeline Status

- Production execution remains the legacy official workflow path.
- The refactored pipeline remains shadow/advisory and is not the production official default unless explicitly enabled.
- Learning, evolution, replay, counterfactual, strategy, and observability components are auxiliary and must not silently take over production decisions.
- Dashboard and status readers are expected to remain read-only and fail-open.
- Submit is not automated by default and compliance decisions remain the user's responsibility.

## Quick Start

```powershell
pip install -r requirements.txt
playwright install chromium
python .\worldquant_auto_workflow.py
```

The default entrypoint opens the Chinese CLI menu for starting the workflow, splitting templates, configuring local authorized settings, and viewing status/logs.

Command-line mode remains available:

```powershell
python .\worldquant_auto_workflow.py split --template-file .\input_templates\test.txt --max-templates 2
python .\worldquant_auto_workflow.py run --use-split --max-templates 2
python .\worldquant_auto_workflow.py status
```

Run tests:

```powershell
python -m pytest
```

## Configuration

- `config.example.json` is the public template.
- Copy it to `config.json` for local use and edit it with authorized local settings.
- `config.json` must not be committed.
- Runtime databases, logs, screenshots, browser state, cookies, credentials, and private Alpha results must not be committed.
- See [docs/CONFIG_REFERENCE.md](docs/CONFIG_REFERENCE.md) for configuration categories.

## Release Artifacts

GitHub Releases should publish a clean zip generated by `tools/build_release.py`.

Release artifacts must not contain runtime state, logs, databases, screenshots, secrets, cookies, browser profiles, private datasets, private Alpha results, or platform-owned data.

Current CI validates source compilation and release artifact hygiene. Full pytest is kept as a local/pre-release diagnostic gate until the existing environment-dependent/core test failures are stabilized in a dedicated change.

Packages are not the primary distribution mechanism for this repository. Future Docker-based deployment, if introduced, may consider GitHub Container Registry.
